get started with Git and Github

May 31,2021

What is Git?

Git is a system which take care of when,who,what was done in the code basically a book keeping mechanism. It is free and open source distributed version control system.

How it Works?

You record the local chnges to your code using a cli tool or using GUI(graphical user interface). When using Git,you make changes to your code files normally as development process. When you have completed the coding or want some screenshots of chnges ,add the file you chnged to a staging area and then commit them to the version history of your project.

Steps to Use Github:

  • Download github desktop
  • Create a rep with name
  • Launch it with visual studio code(vs code)
  • Create a html file(or anyother file)and save it in vs code
  • You can see changes in github desktop
  • Add summary and description and commit to chnges
  • Open the repository page and you can see the chnges made
  • Add an README.md file in vs code save it and commit to the changes
    README.md(describes your repository )

Few Summary details used in day to day work:

  1. feat:new feature
  2. fix:bug fix
  3. docs:chnge in documentation
  4. style:everything related to styling
  5. refactor:code chnges that neither fixes a bug or adds a feature
  6. test:everything realted to testing
  7. chore:updating build tasks,package managr configs etc..

Pushing &Pulling:

Once you push the file it can be found in github.com as you have named your repo.The default setting will be git push origin master.This pushes your code to main branch of your repo. If any chnges happen to a repo that you wish to pull from GitHub to local system,you simply use git pull to see the chnges

Branches:

Branches are used to store the changes made which are yet to complete or in case of team work different people will work on different branch and after completion they will merge it in main branch .
